Designing for touch

Responsive: there is no one true output for the web

Touch: there is no one true input for the web

It's not "just" visual design anymore.

It's industrial design.

When you design a touchscreen interface, you have real ergonomics to consider.

You're designing a physical device.

It matters where you put things, because of the comfort in the users hands.

Therefore, we need to know how people hold these devices.

Important implications

Consider what you want to make super easy vs. what you want to keep out of harms way (e.g. stuff that changes data)

Stacking and pinning controls invites error
